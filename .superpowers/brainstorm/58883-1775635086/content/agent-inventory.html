Agent Inventory — New A2A Architecture

7 agents total: 1 gateway + 1 orchestrator + 3 preprocessing + routing + N execution agents

Gateway Layer

GatewayAgent NEW — extracted from ComprehensiveRouter
Question: "Is this simple or complex?"
How: GLiNER entity classification, keyword rules. No LLM call. <100ms.
Outputs: { complexity: simple|complex, modality: video|text|..., generation_type: raw|summary|report }
Simple path: Maps directly to execution agent (e.g., video modality + raw results → search_agent)
Complex path: Forwards to OrchestratorAgent
Source: GLiNER fast path from routing/router.py ComprehensiveRouter

Orchestration Layer

OrchestratorAgent ENHANCED — gains 5 features from MultiAgentOrchestrator
Question: "What's the plan and execute it?"
How: DSPy LLM planner dynamically decides agent sequence + parallel groups
Features ported from MultiAgentOrchestrator:
  • Streaming (SSE events as each agent completes)
  • Checkpoint/resume (save/restore workflow state)
  • Cross-modal fusion (DSPy ResultAggregatorSignature)
  • Workflow intelligence (learning from past orchestrations)
  • Cancellation (EventQueue-based)
Coordinates via A2A HTTP: Discovers agents via AgentRegistry, calls them over HTTP
